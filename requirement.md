# [cite_start]数据结构课程设计：主题1 - 导航系统任务要求 [cite: 1, 15]

## 1. 系统概述
* [cite_start]**目标用户**：汽车驾驶员 [cite: 18]。
* [cite_start]**核心目标**：设计一款导航软件，不仅能计算到达目的地的最佳路线，还能显示当前位置附近的附加信息（如加油站、餐厅等），以此实现导航系统的核心功能 [cite: 16, 17]。

## 2. 数据配置与模拟数据生成
[cite_start]导航系统中的地图需要被抽象为一个图，其中位置信息对应顶点，路径信息对应边 [cite: 19][cite_start]。您需要设计一个算法来生成模拟的地图数据，具体要求如下 [cite: 20]：
* [cite_start]**顶点生成**：在二维平面上随机生成 N 个顶点（N >= 10000），每个顶点代表地图上的一个位置 [cite: 21]。
* [cite_start]**边与路径建构**：对于每一个位置 x，随机建立几条连接到 x 附近位置的边 [cite: 22][cite_start]。每条边代表一条路径，其长度等于这两个顶点之间的二维坐标距离 [cite: 23]。
* [cite_start]**合理性约束**：生成的模拟数据必须保证最终的图是一个连通图，并且道路之间不能存在不合理的交叉 [cite: 24]。

## 3. 核心功能需求 (Functional Requirements)
[cite_start]您的系统必须实现以下五大核心功能 [cite: 25]：

* **F1. [cite_start]地图显示**：输入一个坐标，系统需显示距离该坐标最近的100个顶点及其关联的边 [cite: 26]。
* **F2. [cite_start]地图缩放**：实现地图的放大和缩小功能 [cite: 28][cite_start]。提示：随着地图缩小，屏幕上会出现更多点导致视觉混乱，需考虑在单位面积内仅显示一个代表性的点 [cite: 28, 29]。
* **F3. [cite_start]最短路径**：指定任意两个位置 A 和 B，计算从 A 到 B 的最短路径，并在界面上显示该路径所经过的顶点和边 [cite: 30]。
* **F4. 交通模拟**：
    * [cite_start]为图中的每条边增加两个属性：容量 v（道路饱和前可容纳的最大车辆数）和当前车辆数 n（当 n>v 时表示道路超载） [cite: 31]。
    * [cite_start]假设道路长度为 L，行驶时间可通过公式 c * L * f(n/v) 来模拟计算，其中 c 为常数 [cite: 32]。
    * [cite_start]f(x) 是一个分段函数：当 x <= 阈值时，f(x)=1；当 x > 阈值时，f(x)=1+e^x [cite: 33][cite_start]。容量 v 和长度 L 是预先定义的固定参数 [cite: 34]。
    * [cite_start]模拟汽车在地图上行驶的状况（为简便起见，假设所有穿过同一道路的汽车花费的时间均为 c * L * f(n/v)） [cite: 35]。
    * [cite_start]实现交通流的动态可视化：在任意给定时间和坐标，需显示附近的所有路径，并用不同的颜色或其他方式动态展现其交通流量级别 [cite: 36]。
* **F5. [cite_start]感知交通的最短路径**：在任何给定的时间，指定位置 A 和 B [cite: 37][cite_start]。计算基于当前交通状况的行驶时间最短的最优路径，并在界面上展示对应的顶点和边 [cite: 37]。
