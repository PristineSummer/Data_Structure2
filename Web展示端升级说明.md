# Web 展示端升级说明

本文档简要说明本项目 Web 展示端相对于原始版本完成的主要修改、新增功能、性能优化与验收方式。升级目标是将原本偏基础演示的 Web 页面，改造成适合课程答辩展示的导航系统：能展示大规模路网、解释算法过程、模拟交通拥堵，并支持现场交互实验。

## 1. 项目升级概览

原始 Web 版以 Flask 提供页面和接口、前端以较简单的地图展示为主。本轮升级后，Web 端已改造为：

- 后端继续使用 Flask 提供 API 和静态资源托管。
- 前端主展示端升级为 React + Vite + TypeScript。
- 地图渲染采用 Leaflet + 多层 Canvas 绘制，减少大量 DOM 节点带来的卡顿。
- 默认展示规模提升到 10000 点，支持缓存与 LOD 降采样。
- 增加算法动画、交通态势、路径解释、事故实验、POI 场景和结果导出等答辩展示能力。

PySide6 桌面版未纳入本轮改造范围，原桌面端行为保持不变。

## 2. 前端展示升级

Web 前端新增并优化了以下展示能力：

- 使用 React/Vite/TypeScript 重构主展示界面，组件化管理操作台、地图、仪表盘、路线卡片和 POI 面板。
- 使用 Canvas 分层绘制道路、热力图、节点、车辆、搜索轨迹和路线，提升大规模地图渲染性能。
- 增加顶部交通 dashboard，展示节点/道路数、活跃车辆、平均拥堵率、拥堵道路数量、拥堵趋势和路况占比。
- 增加右上角全景 overview 缩略图，可显示全图概览、当前视口、路径与事故范围，并支持点击定位。
- 左侧支持自定义生成点数，快捷按钮调整为 `5000 / 10000 / 20000 / 30000`。
- `Top 拥堵道路` 不再常驻地图底部，改为左侧“拥堵道路定位”面板中手动展开，避免挤占地图空间。
- 增强 POI 显示与场景预设，支持加油站、餐厅、停车场、维修、医院等服务点快捷导航。
- 优化 POI 图标显示，例如加油站 `⛽`、餐厅 `🍴`、停车场 `P`、维修 `🔧`、医院 `H`。
- 车辆动画层改为低缩放隐藏、稍微放大后显示，减少全图视角下的信息拥挤。

## 3. 算法与路径能力

本轮升级重点增强了“算法可解释性”和“路径决策展示”：

- 为 A* 和 Dijkstra 增加真实搜索轨迹采集，返回访问节点顺序与成功松弛边。
- 前端支持播放算法搜索动画，可调节播放速度、暂停和重播。
- 新增 A* vs Dijkstra 竞速对比面板，展示访问节点数、耗时、路径长度和访问节点减少比例。
- 新增路径决策解释面板，对比静态最短路和交通感知路径，说明是否避开拥堵路段、节省或增加了多少通行时间。
- 新增交通感知路径展示，路径可根据道路拥堵状态选择绕行路线。
- 新增手动事故实验模式，可设置事故半径和强度，点击地图注入局部拥堵并重新分析路径。
- 新增道路 hover 信息卡，显示道路端点、长度、容量、当前车辆数、拥堵率、拥堵等级和预计通行时间。
- 支持一键导出演示结果，生成 Markdown 报告和 JSON 快照，便于答辩材料整理。

## 4. 后端 API 扩展

Flask API 在原有地图、路径和交通接口基础上进行了扩展：

- `/api/path` 与 `/api/traffic_path` 增加 trace 支持，返回 `path_vertex_ids`、`visited`、`relaxed_edges`、`trace_truncated` 等字段。
- `/api/viewport` 增加道路交通字段，包括长度、容量、当前车辆数、拥堵率、拥堵等级和通行时间。
- `/api/viewport` 增加 LOD 相关能力，支持 `max_vertices`、`max_edges`、`lod` 等参数，返回代表视口和截断信息。
- 新增 `/api/minimap`，为 overview 缩略图提供降采样全图数据。
- 新增 `/api/demo/setup`，支持一键演示流程，自动生成/加载地图、启动交通、选择跨城路线、注入事故并返回对比路径。
- 新增 `/api/analytics/traffic`，返回拥堵等级统计、Top 拥堵道路、历史拥堵趋势、平均/最大拥堵率和活跃车辆数。
- 新增 `/api/poi/categories` 与 `/api/poi/search`，支持按类型搜索附近 POI。
- 新增 `/api/compare_algorithms`，用于同一起终点下对比 A* 与 Dijkstra。
- 新增 `/api/route/explain`，用于返回静态路径与交通感知路径的决策解释。
- 复用 `/api/traffic/inject` 支持手动事故实验。

## 5. 性能与稳定性优化

为支持 30000 点答辩展示规模，完成了以下性能优化：

- Web 端默认点数改为 30000，目标从“基础可运行”提升为“大规模答辩展示”。
- 增加地图缓存机制，按点数、seed、地图尺寸和 POI 密度生成缓存键，缓存文件写入 `data/generated/`，避免同参数重复生成大图。
- 前端根据缩放级别和地图规模自动选择 overview/detail 视口，低缩放时只展示代表道路和重要道路，放大后再加载细节。
- 后端 `/api/viewport` 支持返回数量限制，避免一次性把全量节点和道路传给浏览器。
- 道路 hover 在低缩放或边数过多时自动关闭精确扫描，减少鼠标移动带来的卡顿。
- 交通 Top 拥堵道路使用 top-k 思路优化，避免每次全量排序。
- 大图模式下降低交通轮询频率，减少前后端刷新压力。
- Flask 支持 React 构建产物静态托管，生产期可通过 `python web_server.py` 直接访问 Web 展示端。

## 6. 验收方式

本轮升级过程中使用过以下命令进行验证：

```powershell
python -X utf8 test_web_api.py
```

```powershell
cmd /c npm --prefix web_client run typecheck
```

```powershell
cmd /c npm --prefix web_client run build
```

常用手动验收场景包括：

- 生成 10000 点地图，检查首次生成和缓存加载效果。
- 启动交通模拟，观察 dashboard、热力图、车辆层和 Top 拥堵道路是否正常。
- 运行一键演示，检查事故注入、静态路径、交通感知路径和路径解释。
- 使用 A* vs Dijkstra 面板播放真实搜索轨迹。
- 使用手动事故实验模式点击地图注入拥堵并重新规划路径。
- 使用 POI 场景预设导航到医院、加油站、停车场、维修点或餐厅。
- 导出演示结果，检查 Markdown 和 JSON 文件是否包含关键指标。

## 7. 总结

通过本轮升级，Web 展示端已经从基础导航页面扩展为一个答辩级可交互系统。它不仅能展示地图和路径，还能说明算法搜索过程、解释交通感知路径选择原因、模拟事故拥堵、搜索 POI 服务点，并支持 30000 点规模下的较稳定展示。
